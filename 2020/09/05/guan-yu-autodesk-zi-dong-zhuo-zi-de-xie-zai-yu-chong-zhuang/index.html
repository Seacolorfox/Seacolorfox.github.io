如何在宁的Autodesk（自动桌子）坏掉的情况下卸载或重装Autodesk？（修好自动桌子）

​ 辣鸡Autodesk反人类软件又又又又坏啦！CAD、3dsmax又又又打不开啦，可以不用上学上班啦（划掉

重装也全部是❌❌❌ 我不干活啦！

⭐此 ** 文档中针对的是Autodesk自动桌子产品（如3dsmax、CAD等）在非正常卸载，或者安装失败的情况下的完全卸载**

比如出现以下情况：

  • 尝试升级版本，却一直发生各种奇怪的错误。
  • 安装的时候因为各种因素导致安装终止，致使后续安装无法进行以及安装无限失败。
  • 无法启动。

前两天帮学妹清理电脑发现一堆没卸载完全的自动桌子腿,折腾很久比较脑壳大才折腾好（说好弄一台结果来四台机儿我枯了你是个好人.jpg）

网上有那种付费的卸载工具，虽然简单好用但是似乎有问题 会顺带把注册表给扬了

实际主要思路无非三条：

  • 清理残余组件（桌子腿）。
  • 使用第三方工具完全清除卸载失败组件
  • 清理注册表

1.使用 Autodesk uninstallTool （Autodesk 官方的卸载工具）

​ 如果有安装过Autodesk的软件，一般在开始菜单→Autodesk文件夹中存在这个工具（Uninstall Tool）

​ 其实就是偷懒啦。。。用这个可以直接看到自动桌子装的全部组件。。之后只要用Geek什么的清理卸载失败的或者残留的就可以了。

​ 如果没有的话。。我放个下载链接。。。QWQ

​ 链接：https://pan.baidu.com/s/165_6-2AsjoWfTSnz4Zr94Q
​ 提取码：zibm

​ 解压密码：@Seacolorfox

2.使用卸载工具

⭐如果Autodesk uninstallTool 卸载不完全，请使用卸载工具

例如Geek Uninstaller，官网：https://geekuninstaller.com/

绿色可免费小巧点开即用。

使用Geek勾选未卸载干净的组件，如果不知道需要卸载些什么，打开安装包，点击至开始安装处点击安装详情即可看到会被安装的组件（安装失败的没有被打勾）。

3.清理注册表

⭐1.使用微软官方工具（Microsoft Program Install and Uninstall Troubleshooter）
下载链接：https://support.microsoft.com/en-us/help/17588/windows-fix-problems-that-block-programs-being-installed-or-removed

该疑难解答程序可帮助解决以下问题：

  • 64 位操作系统上的损坏注册表项。
  • 控制更新数据的损坏注册表项。
  • 阻止安装新程序。
  • 阻止完全卸载或更新现有程序。
  • 阻止你通过“控制面板”中的“添加或删除程序”（或“程序和功能”）卸载程序。
  • 如果要删除恶意软件，则可以使用 Windows 安全中心（或其他防病毒工具），也可以使用Windows 恶意软件删除工具。

⭐2.手动清理注册表

  • ⭐若Microsoft Program Install and Uninstall Troubleshooter也没有作用的话，请手动清理注册表。
  • ⭐手动清理注册表请细心细心再细心，避免对windows造成功能性缺失。
  • 以3DSMAX为例：(如果是卸载其他请清除对应的文件夹。。请不要一直欺负3DSMAX)
    删除注册表文件：
  • 点击开始 运行，输入regedit，打开注册表（win10用户直接点左下角小娜搜索“注册表”就可以打开）。
  • 删除：HKEY_CURRENT_USER\software\Autodesk\3ds Max
  • 删除：HKEY_LOCAL_MACHINE\software\Autodesk\3ds Max
  • 删除：HKEY_LOCAL_MACHINE\software\classes\installer\products目录下依次单击文件夹（人工遍历），在右边ProductName的参数可以看到该文件夹对应的安装程序，按需删除。（？不知道删除哪些？，打开Geek或者Autodesk安装包，找到要清除的组件名再在注册表上慢慢找到并删除就可以了，比如材质库组件库blabla）
  • 删除：HKEY_CLASSES_ROOT下的Max文件夹 （没有就别管.jpg）
  • Tips：products目录下注册存放着所有安装程序的安装源路径。（所以请务必细心谨慎）

4.安装

5.许可问题 例如：卡在加载，检查许可，无法初始化，以及20验证错误、安全性验证错误等

Autodesk官方文档给出的解决方案：

仅 当您遇到许可问题时删除许可存储文件果。如果您没有许可问题，请跳过此步骤。以下是最常见的许可问题症状：
删除以下许可（受信任的存储）文件（它们位于不同的位置，具体取决于您的操作系统）。

  • 尝试启动产品时，它停留在“正在检查许可”或“正在加载”步骤。
  • 尝试启动产品时，显示以下错误消息：“无法初始化 adlm: 内部错误消息: <初始化使用跟踪组件时出错。> 错误代码：<-104>”

重要信息：如果您使用的是单机永久使用许可，则删除 adskflex_###_tsf.data 文件可能导致您的软件停止工作。首先检查您的旧版 Autodesk 单机许可软件是否在传输层安全性 (TLS)：需要应用更新以便继续访问软件中列出。如果已列出，请安装更新。然后，您可以安全地删除 adskflex 文件。如果您的软件未列出，则在删除 adskflex 文件后无法重新许可，并且您的软件将停止工作。

Vista 或 Windows 7/8/10 C:\ProgramData\FLEXnet\adskflex_########_tsf.data
C:\ProgramData\FLEXnet\adskflex_########_tsf.data.backup
Windows XP C:\Documents and Settings\All Users\Application Data\FLEXnet\adskflex_########_tsf.data
C:\Documents and Settings\All Users\Application Data\FLEXnet\adskflex_########_tsf.data.backup
macOS /Library/Preferences/FlexnetPublisher/FLEXnet/adskflex_########_event.log
/Library/Preferences/FlexnetPublisher/FLEXnet/adskflex_########_tsf.data
/Library/Preferences/FlexnetPublisher/FLEXnet/adskflex_########_backup 注意：如果您的计算机上没有 Adobe 产品，请也删除以下文件夹：/Library/Application Support/Flexnet Publisher

⭐如果是学习版或者能够再认证的话。。。

​ 直接将FLEnet文件夹移除即可（一 步 到 位）

​ 💢请谨慎使用！！！有一定几率导致远古版本出现无法激活的问题（无限安全锁错误，需要重装软件）

​ 路径：C:\ProgramData\FLEnet

​ Tips：ProgramData在windows中默认隐藏，请确认对隐藏项目可见。

6.其他步骤（非必要）

以下是我贴的官方文档
实际上在1、2步骤我们的工具基本上已经清理好了,所以请选择性清理

  1. 删除所有剩余的残留文件。请勿清空文件夹。仅删除与您要卸载的产品相关的那些文件。
Windows 产品 C:\Program Files\Autodesk\
C:\ProgramData\Autodesk\
C:\Users\YourUserid\AppData\Local\Autodesk\
C:\Users\YourUserid\AppData\Roaming\Autodesk\
C:\Users\YourUserid\AppData\Roaming\Autodesk\ADUT（如果适用）
Mac 产品 Library\Application Support\Autodesk
Library\Application Support\Preferences
  1. 清除 Windows TEMP 文件。
  1. 从安装程序中，重新尝试安装 Autodesk 产品。
  1. 如果安装仍然失败

您可能需要删除与所有 Autodesk 产品和支持软件（如附加模块）相关的文件。

  • 有关适用于 Windows 的此过程的完整详细信息，请参见[删除 Autodesk 软件 (Windows)](https://knowledge.autodesk.com/zh-hans/customer-service/download-install/remove-products/remove-windows-products#Uninstall peripheral applications.)。
  • 对于 Mac 产品，使用 Spotlight 搜索所有 Autodesk 软件和支持文件，然后将其移到垃圾桶。
